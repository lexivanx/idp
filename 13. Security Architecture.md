## 13. Security Architecture

### 13.1 Authentication Flow

```
User → Browser → Azure AD (OIDC) → ID Token + Access Token → CloudMultiPloy API
                                                                    │
                                                            Token Validation
                                                           (JWKS endpoint)
                                                                    │
                                                            RBAC Check
                                                         (Identity Service)
                                                                    │
                                                             Authorized
                                                            API Response
```

### 13.2 Secret Management

```
Azure Key Vault
    │
    ├── Secrets Store CSI Driver (mounted as volumes in pods)
    │
    ├── External Secrets Operator (syncs to Kubernetes Secrets)
    │
    └── Terraform (reads/writes during provisioning)

Secret Rotation:
    1. Azure Key Vault auto-rotation (certificates, keys)
    2. CronJob-based rotation for database credentials
    3. Operator-managed rotation for service-to-service tokens
```

### 13.3 Network Security

- **Kubernetes Network Policies**: Default-deny in all team namespaces; explicit allow rules per service.
- **Istio Authorization Policies**: Service-to-service access control based on mTLS identity.
- **Azure NSGs**: Restrict traffic between subnets (AKS, databases, private endpoints).
- **Private Endpoints**: All Azure PaaS services (PostgreSQL, Redis, Service Bus, Key Vault) accessible only via private endpoints — no public endpoints.
- **Azure Private DNS Zones**: Resolve private endpoints via internal DNS.

### 13.4 Container Security

```
Build Phase:
    ├── SAST: SonarQube / Semgrep
    ├── Dependency scanning: Dependabot / Snyk
    ├── Container scan: Trivy (in pipeline)
    └── Image signing: Cosign (Sigstore)

Runtime Phase:
    ├── Admission controller: OPA Gatekeeper
    │   ├── No privileged containers
    │   ├── No root users
    │   ├── Required resource limits
    │   ├── Approved registries only
    │   └── Required labels/annotations
    ├── Runtime scanning: Falco
    └── Pod Security Standards: restricted
```
