## 15. GitHub Repository Structure

### 15.1 Multi-Repo Strategy

The project uses a multi-repository strategy to mirror real-world organizational patterns and demonstrate repository management skills:

```

│
```

### 15.2 Repository Standards (Applied to Every Repo)

Every repository must contain:

```
repo-root/
├── README.md                    # Purpose, setup, usage, architecture decisions
├── CONTRIBUTING.md              # How to contribute, code style, PR process
├── CHANGELOG.md                 # Keep a changelog (semver)
├── LICENSE                      # Apache 2.0
├── Makefile                     # Standard targets: build, test, lint, run, docker-build
├── Dockerfile                   # Multi-stage build
├── .dockerignore
├── .gitignore
├── .editorconfig                # Consistent coding style
├── .github/
│   ├── workflows/
│   │   ├── ci.yaml              # Lint + test + build
│   │   ├── release.yaml         # Semantic release
│   │   └── security.yaml        # Dependency scanning
│   ├── ISSUE_TEMPLATE/
│   │   ├── bug_report.md
│   │   ├── feature_request.md
│   │   └── task.md
│   ├── PULL_REQUEST_TEMPLATE.md
│   └── CODEOWNERS
├── docs/
│   └── ...
└── api/                         # OpenAPI spec (for services)
    └── openapi.yaml
```
