## 17. Definition of Done (DoD)

A user story is considered "Done" when ALL of the following are met:

**Code Quality**:
- [ ] Code compiles and passes all linters (golangci-lint for Go, ESLint for TypeScript, Checkstyle/SpotBugs for Java, ruff/mypy for Python)
- [ ] Unit tests written and passing with >80% coverage for business logic
- [ ] Integration tests written for API endpoints
- [ ] No CRITICAL or HIGH security vulnerabilities in dependencies
- [ ] Code reviewed and approved by at least one peer (PR review)

**Documentation**:
- [ ] README updated if the change affects setup, configuration, or usage
- [ ] OpenAPI spec updated if API contracts changed
- [ ] ADR written if a significant architectural decision was made
- [ ] Inline code comments for complex logic
- [ ] CHANGELOG updated

**Infrastructure**:
- [ ] Dockerfile builds successfully
- [ ] Helm chart validates (`helm lint`, `helm template`)
- [ ] Terraform validates (`terraform fmt`, `terraform validate`, `terraform plan`)
- [ ] ArgoCD Application syncs successfully in dev environment
- [ ] Network policies applied and tested

**Observability**:
- [ ] Prometheus metrics instrumented
- [ ] ServiceMonitor configured
- [ ] Grafana dashboard updated (if applicable)
- [ ] Alert rules configured (if applicable)
- [ ] Structured logging with correlation IDs

**Deployment**:
- [ ] CI pipeline passes (lint, test, build, scan)
- [ ] Container image pushed to ACR
- [ ] Deployed to dev environment via ArgoCD
- [ ] Smoke test passed in dev environment
