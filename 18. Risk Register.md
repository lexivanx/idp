## 18. Risk Register

| ID | Risk | Probability | Impact | Mitigation |
|----|------|-------------|--------|------------|
| R-001 | Azure cost overruns during development | High | Medium | Use dev-sized resources, implement auto-shutdown for non-prod, set Azure cost alerts at $50, $100, $200 thresholds |
| R-002 | Terraform state corruption | Low | Critical | State locking, state versioning, regular state backups, never run Terraform outside of the pipeline/provisioner service |
| R-003 | Kubernetes complexity leads to debugging overhead | Medium | Medium | Invest in observability early (Sprint 0), create troubleshooting runbooks, use k9s for local debugging |
| R-004 | Scope creep — too many features for MVP | High | High | Strict MoSCoW prioritization, ruthless backlog grooming, defer multi-cloud to post-MVP |
| R-005 | Solo development bottleneck | High | Medium | Automate everything possible, use code generation, prioritize working software over perfection |
| R-006 | Azure AD configuration complexity | Medium | Medium | Follow Microsoft documentation exactly, create ADR for auth decisions, test early |
| R-007 | Service mesh overhead (Istio resource consumption) | Medium | Low | Right-size Istio installation (disable unused features), consider sidecar-less mode |
| R-008 | Database performance under load | Low | Medium | Connection pooling (PgBouncer), query optimization, read replicas, load testing |
