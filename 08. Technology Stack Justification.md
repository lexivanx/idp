## 8. Technology Stack Justification

| Layer | Technology | Justification |
|-------|-----------|---------------|
| **Backend (Core Services)** | Go 1.22+ | Excellent concurrency model (goroutines), fast compilation, small binary size ideal for containers, strong standard library for HTTP/gRPC, dominant language in cloud-native ecosystem (Kubernetes, Docker, Terraform are all Go). Used for: Catalog, Provisioner, Pipeline, Identity, Notification, Template Engine, Audit services. |
| **Backend (Ticket & Workflow)** | Java 21 + Spring Boot 3.2 | Spring Statemachine provides a battle-tested workflow engine far more mature than Go alternatives; Spring Security for fine-grained ticket authorization; Spring Data JPA for complex relational queries on workflow state; Bean Validation for request validation; virtual threads for high-throughput ticket operations. |
| **Backend (Cost Service)** | Python 3.12 + Django 5 + Django REST Framework | Python's data processing ecosystem (pandas, NumPy) is unmatched for cost aggregation, trend analysis, and anomaly detection; Django ORM handles complex cost reporting queries cleanly; Django REST Framework provides rapid API development with serializers, viewsets, and auto-generated browsable API; celery + django-celery-beat for scheduled cost data collection tasks. |
| **Backend (Template Engine)** | Go `text/template` + custom | Aligns with Helm's templating model; keeps the stack homogeneous. |
| **Backend (Golden Path Template)** | Java 21 + Spring Boot | Used for the Java Spring Boot API golden path template; demonstrates polyglot support and validates the Java CI/CD pipeline path. |
| **Frontend** | React 18+ / Next.js 14+ / TypeScript 5+ | Industry-standard SPA framework; TypeScript provides type safety; Next.js provides SSR, API routes, and file-based routing. |
| **CLI** | Go + Cobra | Same language as backend; single binary distribution; cross-platform; follows established patterns (kubectl, gh). |
| **Primary Database** | PostgreSQL 16 | ACID compliance, JSONB for flexible metadata, excellent performance, mature ecosystem, managed offering on Azure (Flexible Server). |
| **Cache** | Redis 7 | Session storage, API response caching, rate limiting (token bucket), pub/sub for real-time notifications. |
| **Message Broker** | Azure Service Bus | Managed, reliable message delivery; supports topics/subscriptions for event-driven architecture; dead-letter queue for failed messages; native Azure integration. |
| **Container Orchestration** | AKS (Azure Kubernetes Service) | Managed Kubernetes; integrates with Azure AD, Azure Monitor, Azure Policy; auto-scaling; multiple node pools. |
| **Service Mesh** | Istio | mTLS between services, traffic management (canary, circuit breaking), observability (distributed tracing), access policies. Selected over Linkerd for richer feature set and wider adoption. |
| **Infrastructure as Code** | Terraform 1.7+ | Multi-cloud support (critical for post-MVP extensibility), declarative, state management, huge provider ecosystem, HCL is readable. |
| **CI/CD** | Azure DevOps Pipelines | Native Azure integration, YAML-based pipelines, approval gates, variable groups, service connections. |
| **GitOps** | ArgoCD | Kubernetes-native, declarative, auto-sync with drift detection, multi-cluster support, excellent UI for deployment visualization. |
| **Monitoring** | Prometheus + Grafana | Industry standard for Kubernetes monitoring; PromQL is powerful; Grafana provides rich dashboarding; both are cloud-agnostic. |
| **Alerting** | Alertmanager | Native Prometheus integration, routing tree, silencing, grouping, inhibition rules. |
| **Cloud Monitoring** | Azure Monitor + Log Analytics | Native Azure resource monitoring, KQL for log queries, cost included with Azure resources. |
| **Configuration Mgmt** | Ansible | Used for AKS node customization, golden image configuration, and operational runbook automation. |
| **Container Runtime** | Docker | Industry standard; BuildKit for efficient multi-stage builds; compatible with AKS. |
| **Helm** | Helm 3 | Kubernetes package manager; used for deploying platform components and golden path service charts. |
| **Scripting** | Bash / Python / Go | Bash for simple automation; Python for complex scripting (Terraform wrapper, data processing); Go for performance-critical tooling. |
