## 9. Infrastructure Architecture

### 9.1 Azure Infrastructure (MVP)

```
Azure Subscription: cloudmultiploy-production
├── Resource Group: rg-cloudmultiploy-platform
│   ├── AKS Cluster: aks-cloudmultiploy-prod
│   │   ├── System Node Pool: 3x Standard_D4s_v5 (platform components)
│   │   ├── User Node Pool: 3-20x Standard_D8s_v5 (workloads, autoscaling)
│   │   └── Monitoring Node Pool: 2x Standard_D4s_v5 (Prometheus, Grafana)
│   ├── Azure Container Registry: acrcloudmultiploy
│   ├── Azure Key Vault: kv-cloudmultiploy-prod
│   ├── Azure DNS Zone: cloudmultiploy.company.com
│   ├── Virtual Network: vnet-cloudmultiploy-prod
│   │   ├── Subnet: snet-aks (AKS pods, /16)
│   │   ├── Subnet: snet-db (PostgreSQL, /24)
│   │   └── Subnet: snet-endpoints (Private endpoints, /24)
│   ├── Azure Application Gateway + WAF v2 (Ingress controller)
│   ├── Log Analytics Workspace: law-cloudmultiploy-prod
│   └── Azure Monitor (metrics, alerts)
│
├── Resource Group: rg-cloudmultiploy-data
│   ├── Azure Database for PostgreSQL (Flexible Server)
│   │   ├── Primary: 4 vCores, 16GB RAM, 256GB storage
│   │   └── Read Replica: 4 vCores (for reporting queries)
│   ├── Azure Cache for Redis (Premium P1)
│   └── Azure Service Bus (Standard tier)
│
├── Resource Group: rg-cloudmultiploy-state
│   ├── Storage Account: stcloudmultiploystate (Terraform state)
│   └── Table Storage: (Terraform state locking)
│
├── Resource Group: rg-cloudmultiploy-monitoring
│   └── (Prometheus/Grafana data, managed disks)
│
└── Resource Group: rg-cloudmultiploy-networking
    ├── Azure Front Door (global load balancing, CDN)
    ├── Private DNS Zones
    └── Network Security Groups
```

├── cloudmultiploy-operators           # Custom Kubernetes operators

### 9.3 Network Architecture

```
Internet
    │
    ▼
Azure Front Door (CDN + WAF + Global LB)
    │
    ▼
Azure Application Gateway (Regional LB + SSL termination)
    │
    ▼
Istio Ingress Gateway (Kubernetes Ingress)
    │
    ▼
Istio Virtual Services (routing rules)
    │
    ├─── cloudmultiploy.company.com → web-portal
    ├─── api.cloudmultiploy.company.com → API Gateway
    ├─── argocd.cloudmultiploy.company.com → ArgoCD UI
    ├─── grafana.cloudmultiploy.company.com → Grafana
    └─── *.dev.cloudmultiploy.company.com → Ephemeral envs

Internal (mTLS via Istio):
    catalog-service ←──mTLS──→ provisioner-service
    ticket-service  ←──mTLS──→ notification-service
    All services    ←──mTLS──→ All services
```
