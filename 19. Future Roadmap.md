## 19. Future Roadmap — Post-MVP

### Phase 2: Multi-Cloud (AWS)
- Implement AWS adapter (EKS, RDS, ElastiCache, SQS, S3)
- Create Terraform modules for AWS resources
- Implement cloud provider selection in service creation wizard
- Implement cross-cloud service mesh connectivity (Istio multi-cluster)

### Phase 3: Multi-Cloud (GCP)
- Implement GCP adapter (GKE, Cloud SQL, Memorystore, Pub/Sub, Cloud Storage)
- Create Terraform modules for GCP resources

### Phase 4: Advanced Features
- **Developer Portal (Backstage Integration)**: Evaluate Backstage as a frontend alternative
- **GitOps with FluxCD**: Offer FluxCD as an alternative to ArgoCD
- **Linkerd Option**: Offer Linkerd as a lightweight service mesh alternative
- **Custom Operator Marketplace**: Allow teams to publish and share custom operators
- **FinOps Advanced**: Reserved instance recommendations, spot instance management, cost anomaly detection
- **AI-Assisted Troubleshooting**: LLM-powered log analysis and root cause suggestion
- **Service Mesh Observability**: Advanced traffic analysis, circuit breaker dashboards
- **Multi-Cluster Management**: Manage services across multiple Kubernetes clusters
- **Database Migrations Management**: Track and execute schema migrations via the platform
- **Cassandra and MongoDB Support**: Add NoSQL golden path templates
- **MS SQL Support**: For organizations with .NET workloads

### Phase 5: Enterprise Features
- Multi-tenancy across organizations
- SSO with SAML in addition to OIDC
- Advanced compliance frameworks (SOC 2, HIPAA, PCI-DSS)
- Custom approval chain builder (UI-based workflow editor)
- Mobile companion app (React Native)
